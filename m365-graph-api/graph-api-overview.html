Microsoft 365 APIs for Newbie Developers

This corner is for newbie Microsoft 365 Developers. Especially those who are interested in learning the basics of creating simple apps that extend the Microsoft 365 functionality by making use of Microsoft 365 APIs. Here you will find everything you need to get started with Microsoft 365 APIs like how to register your app, how to generate the API key, how to create app permissions and get the Microsoft 365 admin consent for it etc.,

Note: Keep an eye on this page for new articles every week. If articles don't load, this means they are still under wraps

  • Introduction - What are Microsoft 365 APIs? How you can use them?
  • How to Create a Microsoft 365 Developer account?
  • How to Register your App with Microsoft?
  • How to Generate the Microsoft 365 API Key?
  • How to Create Access Permissions for your App?
  • How the Microsoft 365 tenant admin consents to your App Permissions?
  • How to Authenticate your app and get the access token?
  • Introducing Graph API for handling Microsoft 365 Data
  • Making Your First Microsoft 365 Graph API Call Using Graph Explorer
  • Using Microsoft Authentication Library (MSAL) for app authorization/authentication
  • Exploring Microsoft 365 or Graph API Endpoints

Note: If you remember, the link that led you to this article was titled ‘Graph API for newbie Microsoft 365 developers’. That’s because previously Microsoft 365 API services were split into different modules and you had reach out to each of these modules to understand and use them. With the introduction of Graph API technology, all these Microsoft 365 API services were unified and made one. So, Graph API is now the umbrella term and single point of contact for different API services offered within Microsoft 365.
